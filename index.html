Install the Matlab2016b on Ubuntu10.10

After download the Matlab2016b for Linux and you can see the files below:


R2016b_glnxa64_dvd1.iso
R2016b_glnxa64_dvd2.iso
Matlab 2016b Linux64 Crack.rar

In the case, I put my download file in /home/g214/matlab2016_iso. To extract the .rarfile, you can install the rar by using the command:
$ sudo apt-get install rar

And then decompress by typing:
$ rar x Matlab 2016b Linux64 Crack.rar

In the file Matlab2016bLinux64Crack.rar, the documentreadmeis the one you must read carefully, in which provides two modes of installation and the installing keys. The license_standalone.lic and the license_server.lic are the active files, and the lest two files including some files you need to exchange after installing the Matlab2016b.

  • Mount the .isofile

    In this case, I mount my .isofile in the /home/g214/matlab2016b, note that you just need to mount R2016b_glnxa64_dvd1.isofirst,R2016b_glnxa64_dvd2.isois no need now, and say a word more, you also shouldn’t to decompress the .isofile.

To mount the R2016b_glnxa64_dvd1.iso in /home/g214/matlab2016b, use command:


$ mkdir /home/g214/matlab2016b
$ sudo mount -t auto -o loop /home/g214/matlab2016_iso/R2016b_glnxa64_dvd1.iso  /home/g214/matlab2016b

  • Installation

    Enter the /home/g214/matlab2016b by
    $ cd /home/g214/matlab2016b
    you can see a install file. Return to the higher level directory /home/g214by
    $ cd ..
    and then:
    $ /home/g214/matlab2016b/install

The installation interface will be open.Then you can follow the prompts to install. In this process, you can set and need to remember where the Matlab you want to ianstall. In this case, I install my Matlab2016b in the directory /home/g214/MATLAB/R2016b.

While the R2016b_glnxa64_dvd1.iso is installed completely(about 80% of the whole installation), it will prompt you to inset the second dvd, What you need to do is opening a new terminal and mount the R2016b_glnxa64_dvd2.iso. Note that do not mount the second dvd in the prvious terminal.

To mount the R2016b_glnxa64_dvd2.iso to the directory /home/g214/matlab2016b you can:


$ sudo mount -t auto -o loop /home/g214/matlab2016_iso/R2016b_glnxa64_dvd2.iso /home/g214/matlab2016b

Beacause it is the first time you run this, The Activation Dialog will appears. Chosing the activation offline and the activate file in the /home/g214/matlab2016_iso/Matlab2016bLinux64Crack,which depend on the installation way you use.In this case I use license_standalone.lic.
    
$ cd /home/g214/MATLAB/R2016b/bin # Enter the installation file

$ sudo sourse /etc/environment

How to use Hexo to build and deploy you bolg

Setup

Initial Hexo in target <folder>


$ hexo init <folder>
$ cd <folder>
$ npm install

Creates a new article


 $ hexo new [layout] <title>

Generates static files


 $ hexo generate
Optional: Deploy after generation finishes:``$ hexo generate -d`` Watch file changes:``$ hexo generate -w`` ### Publish 

 $ hexo publish [layout] <filename>

Starts a local server

By default, this is athttp://localhost:4000/


 $ hexo deploy

Deploys your website


$ hexo deploy
If missing Error: 

ERROR Deployer not found: git
You can add something after github update: 

npm install hexo-deployer-git --save
